# 외부 재게재 — 공식 API 자동 게시 폐기, export-only 수동 게시
- 날짜: 2026-06-27
- 결정: Medium·Velog·Tistory 외부 배포는 **붙여넣기용 산출물(export) 자동 생성 + 게시는 전부 수동**. 플랫폼 API로 자동 게시하지 않는다.
- 이유(실측):
  - **Tistory**: Open API 글쓰기 API가 2024년 초 완전 종료. 공식 게시 경로 없음.
  - **Medium**: 2025-01-09부터 신규 integration token 발급 중단. 2025 이전 토큰만 동작 — 우리는 미보유. 사실상 폐쇄.
  - **Velog**: 공식 쓰기 API 자체가 없음. GraphQL 백엔드는 로그인 쿠키(JWT) 기반 비공식·불안정.
  - → REST API 자동 게시 라인은 셋 다 불가능. 남는 현실 경로는 (A)브라우저 반자동, (B)export 후 수동뿐.
  - 프로젝트 원칙("초안 생성까지만 자동, 게시는 사람 확인 후")과 가장 정합하고, 인증·세션 깨짐 보수 부담이 없는 **(B)export-only**를 채택.
- 폐기 옵션:
  - **플랫폼 REST API 자동 게시**: 위 사유로 셋 다 불가/금지.
  - **(A) 브라우저 자동화(Claude-in-Chrome)**: 가능하나 로그인 세션·셀렉터 깨짐 보수 비용이 크고 게시 원칙과 거리. 추후 안정 플랫폼 한정 점진 도입 여지만 남김(하이브리드는 보류).
  - **velog GraphQL write 비공식 호출**: 쿠키 토큰 기반·약관 위험·깨지기 쉬움 → 배제.
