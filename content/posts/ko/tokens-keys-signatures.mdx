## 1. 한 줄 위협
요청을 위조당하면 서버는 "이 요청이 진짜 그 사람에게서 왔는가"를 증명할 수 없다.

## 2. 개념
토큰은 출입증, 키는 도장, 서명은 도장이 찍힌 봉인이다. 셋은 _증명하는 대상_ 이 다르다.

## 3. 취약한 코드
```ts
// 토큰만 보고 통과 — 본문이 바뀌어도 알 수 없다
if (req.headers["x-token"] === SECRET) accept(req);
```

## 4. 공격 원리
토큰은 "누구"만 증명한다. 본문 무결성은 증명하지 않으므로 중간자가 페이로드를 바꿔도 통과한다.

## 5. 방어 구현
```ts
// 본문에 대한 HMAC 서명 — 본문이 바뀌면 서명이 깨진다
const sig = hmacSHA256(SECRET, rawBody);
if (timingSafeEqual(sig, req.headers["x-signature"])) accept(req);
```

| Decision | Why | Rejected |
|---|---|---|
| 본문 HMAC 서명 | 신원 + 무결성을 한 번에 증명 | 토큰 단독: 본문 변조 무방비 |
| 상수시간 비교 | 타이밍 누출 차단 | `===`: 비교 시간이 정보를 흘림 |

## 6. 실전 체크리스트
- [ ] **접근통제(access)**: 서명이 본문 전체를 덮는가
- [ ] 비교는 상수시간인가
- [ ] 비밀은 환경변수·KMS에 있는가
